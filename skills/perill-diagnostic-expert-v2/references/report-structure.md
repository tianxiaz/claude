- Report title (主标题 + 副标题)

- Team size description
- Response count description

### 1.2 Core Findings Summary (核心发现摘要)
3-5 key bullet points summarizing:
- Lowest scoring dimensions
- Largest gaps (current vs expected)
- Most significant blindspots
- Key role differences

### 1.3 Core Diagnostic Conclusion (核心诊断结论)
**Format: 3 logically connected sentences**

1. **Sentence 1 (Intervention Strategy Overview):** Describe the lever dimension(s) and the goal of the intervention strategy.
   - Example: "以 E（外部）与 I（内部）为起点，是利用团队最大盲点（外部协作薄弱）与最低能力（内部流程僵化）作为杠杆，重构其运营系统。"

2. **Sentence 2 (Expected State After Intervention):** Describe the expected state across all dimensions after intervention succeeds, using PERILL dimension abbreviations.
   - Example: "当 E 与 I 对齐后，P（目的）锚定共同方向，R（关系）信任自然浮现并建设性地服务于系统，L（学习）形成可持续的反思改进循环，Ld（领导力）从控制转向赋能，整个团队实现从'被动执行'到'主动协作'的转变。"

3. **Sentence 3 (Insightful Conclusion):** A thought-provoking statement revealing the underlying logic.
   - Example: "组织问题不是通过绘制新结构解决，而是通过教练团队共创跨职能决策权、共享目的与外部问责节奏来化解——流程是关系的载体，关系是领导力的镜子，领导力是学习的催化剂。"

## Section 2: Six-Dimension Deep Diagnosis (六维深度诊断)

For each dimension (P, E, R, I, L, Ld), include:

### Structure per dimension:
- **Dimension name** (English + Chinese)
- **Scores**: Team current / Stakeholder current / Expected
- **Advantage evidence — Team background info** (2 bullet points from background PDF/document)
- **Advantage evidence — Questionnaire data support** (2 bullet points with specific question scores showing strengths, e.g., Q15=4.00)
- **Advantage analysis** (2-3 sentences; first sentence identifies the team's specific strengths in this dimension connected to evidence; subsequent sentences explain the positive significance of these advantages for the team's future development as a system)
- **Factual evidence — Team background info** (3 bullet points from background PDF/document)
- **Factual evidence — Questionnaire data support** (3-4 bullet points with specific question scores, e.g., Q13=2.50)
- **Core problem summary** (2-3 sentences, specific to team's situation)
- **System impact analysis** (2-3 sentences about impact on team's future development as a system)
- **Diagnostic conclusion** (10 words or less, in「」brackets)

### Content guidance per dimension:
**P — Purpose**: Focus on mission clarity, goal alignment, value connection
**E — External**: Focus on stakeholder relations, resource access, market awareness
**R — Relationship**: Focus on trust, psychological safety, conflict handling
**I — Internal**: Focus on processes, decision-making, role clarity, communication
**L — Learning**: Focus on reflection habits, development plans, feedback culture
**Ld — Leadership**: Focus on leadership distribution, leader-team dynamics

## Section 3: Systemic Interconnection Analysis (系统性关联分析)

### 3.1 Dimension Interconnection Analysis

**Format for each interconnection pair:**
```
Dimension Pair: [e.g., E ↔ I]
Causal Overview: [brief description, e.g., "外部协作与内部流程：输入-执行闭环"]

[Analysis in flowing prose]:
[Closed-loop problem description] → [Team system impact] → [Intervention to break the loop]
```

**Key requirement:** Analysis text must be natural, flowing prose (not bullet points), explaining:
1. The closed-loop problem between the two dimensions
2. Its impact on the team as a whole system
3. The intervention approach to break the loop

**Six dimension pairs to analyze:**
1. E ↔ I (外部协作与内部流程：输入-执行闭环)
2. R ↔ I (关系信任与内部授权：安全-责任闭环)
3. I ↔ L (内部流程与学习反思：执行-复盘闭环)
4. P ↔ R (共同目的与关系信任：愿景-凝聚闭环)
5. Ld ↔ I (领导风格与内部授权：控制-依赖闭环)
6. E ↔ L (外部市场与学习发展：需求-适应闭环)

### 3.2 Current Causal Cycle (现状因果循环)

**Format:** Use ONLY PERILL dimension abbreviations串联 (no question scores).

Example:
```
现状因果循环：僵化-控制循环

循环构成路径：
领导力缺乏（Ld）→ 内部系统与流程僵化（I）→ 外部协作缺失（E）→ 团队学习停滞（L）→ 团队成员间关系薄弱（R）→ 团队宗旨目的模糊（P）→ 领导更需进一步强化控制（Ld）
```

Key characteristics paragraph (1-2 sentences).

### 3.3 Intervention Causal Cycle (干预因果循环)

**Format:** Use dimension abbreviations with brief action descriptions between transitions.

Example:
```
干预因果循环：赋能-信任循环

循环构成路径：
从关系破冰（R）→ 建立心理安全感 → 内部开始授权（I）→ 成员主动承担责任 → 外部协作改善（E）→ 市场感知提升 → 目的聚焦（P）→ 愿景共识形成 → 学习反思启动（L）→ 学习机制建立 → 领导风格转型（Ld）→ 赋能型领导 → 关系信任深化（R）
```

Key characteristics paragraph (1-2 sentences).

## Quality Control Checklist

Before finalizing report:
- [ ] All conclusions supported by questionnaire data
- [ ] No inference beyond data scope
- [ ] Distinguish "data fact" from "reasonable inference"
- [ ] Balance team self-assessment and stakeholder perspectives
- [ ] Balance current problems and future development
- [ ] Include both diagnosis and constructive recommendations
- [ ] Section 1.3 and 4.1 contain identical 3-sentence conclusion
- [ ] Six-dimension deep diagnosis includes BOTH background AND questionnaire evidence
- [ ] Six-dimension deep diagnosis includes advantage analysis with BOTH background AND questionnaire evidence
- [ ] Dimension interconnection analysis uses flowing prose format
- [ ] Causal cycles use only dimension abbreviations (no question numbers)
- [ ] Phase intervention goals focus on team as system, not individuals
- [ ] Bar chart is NOT included (redundant with radar chart)
