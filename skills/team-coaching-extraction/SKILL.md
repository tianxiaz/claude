# 团队教练信息提取技能

以 EMCC/ICF 双认证大师级团队教练专家的专业视角，系统性地从团队教练相关材料中提取关键信息，输出带精确溯源的高质量 Markdown 分析报告。

## 核心原则

- **绝对客观**：零篡改、零杜撰、零演绎，所有提取必须有原文依据
- **逐字溯源**：附录中逐字引用原文片段，标注精确来源位置
- **教练视角**：以系统视角和非评判性语言进行分析
- **原子化事实**：复杂事件拆分为独立事实，多方观点全部记录

## 工作流程

执行以下 5 个步骤完成信息提取：

### 步骤 1：输入材料预处理

1. 确定用户提供的材料格式和数量
2. 若材料为 docx/pdf，运行 `scripts/parse_input.py` 解析为统一文本：
   ```bash
   # 单个文件
   python scripts/parse_input.py ./materials.docx -o parsed.txt

   # 整个目录
   python scripts/parse_input.py ./materials_folder/ -r -o parsed.txt
   ```
3. 若材料为 txt/md，直接读取
4. 若解析失败，使用工具的 read_file 能力直接读取
5. 记录每个来源文件的名称，用于后续溯源标注

### 步骤 2：阅读参考框架

阅读以下参考文件，建立提取规范：

- **`references/extraction-framework.md`** — 四大维度的详细提取规范、事实记录格式、溯源要求
- **`references/coach-standards.md`** — EMCC/ICF 能力标准、教练视角分析框架、非评判性表述规范
- **`references/report-templates.md`** — 报告模板结构、引用标注规范、文件命名规范

### 步骤 3：四维度信息提取

按照以下顺序系统提取信息，逐条标注来源引用 `[^编号^]`：

**维度一：团队背景信息**
- 阅读 `extraction-framework.md` 中的"维度一"规范
- 提取企业背景、团队背景、团队现状
- 区分事实陈述与主观评价

**维度二：引入团队教练的目的**
- 阅读 `extraction-framework.md` 中的"维度二"规范
- 提取发起人信息、发起目的、期望解决的问题、利益相关者
- 严格区分发起人原话与转述

**维度三：主要事实**
- 阅读 `extraction-framework.md` 中的"维度三"规范
- 以 5W1H 格式提取每条事实
- **重点提取**不同人的具体行为、言论、观点
- 将复杂事件原子化，多方观点全部记录

**维度四：关系与矛盾发现**
- 阅读 `extraction-framework.md` 中的"维度四"规范
- 基于已提取事实进行系统性分析
- 每个发现标注支持性事实编号
- 区分可直接观察的与推断的（后者标注"推断"）
- 使用 `coach-standards.md` 中的非评判性表述

### 步骤 4：生成主报告

使用 `assets/report-template.md` 作为基础，生成主报告文件 `[team-name]_extraction_report.md`：

1. 填充四个维度的提取内容
2. 确保每一处提取都有 `[^编号^]` 引用标注
3. 使用非评判性教练语言
4. 第五部分"教练视角总结"中，明确标注推断内容

### 步骤 5：生成溯源附录

生成独立的附录文件 `[team-name]_extraction_report_appendix.md`：

1. 创建引用索引表（编号、提取概括、逐字原文、来源文件、来源位置）
2. 确保附录中的原文引用与输入材料逐字一致
3. 确保报告中每一处 `[^编号^]` 都在附录中有对应条目
4. 两个文件保存到 `/mnt/agents/output/` 目录

## 关键规范速查

### 事实记录格式

```markdown
### 事实 [编号]

- **时间**：[原文引用]
- **地点**：[原文引用]
- **人物**：[原文引用]
- **起因**：[原文引用及概括]
- **经过**：[原文引用及概括]
- **结果**：[原文引用及概括]

**关键行为/观点**：
> [逐字引用原文片段] [^编号^]
```

### 分析发现格式

```markdown
#### [发现标题]

[基于事实的发现描述，使用教练视角语言]

**支持事实**：[事实编号列表]
```

### 引用标注格式

- 文内使用：`[^1^]`、`[^2^]`，编号连续
- 附录表格：编号、提取概括、逐字原文、来源文件、来源位置

### 文件命名

- 主报告：`[team-name]_extraction_report.md`
- 附录：`[team-name]_extraction_report_appendix.md`

## 质量检查清单

报告生成后，执行以下检查：

- [ ] 四个维度全部覆盖，无遗漏
- [ ] 所有提取内容在输入材料中有明确依据
- [ ] 每一处 `[^编号^]` 在附录中都有对应条目
- [ ] 附录中的原文引用与输入材料逐字一致
- [ ] 事实原子化，多方观点全部记录
- [ ] 分析发现基于事实，标注支持性事实编号
- [ ] 推断内容明确标注为"推断"
- [ ] 使用非评判性教练语言
- [ ] 人名、角色等关键信息前后统一
- [ ] Markdown 格式规范，排版整洁
