# PERILL团队诊断与发展建议生成

## 概述

本技能用于基于PERILL团队诊断模型的评估数据，生成面向被教练团队的高质量诊断分析与发展建议报告。输出包括PERILL六维度深度诊断分析、PERILL模型整体综合诊断分析，以及配合团队教练干预策略的团队发展建议。

**核心特色：**
- 诊断分析更深入、洞察更深刻
- 语言表达更流畅、可读性更强
- 建议更具体、可操作性更高
- 系统视角贯穿始终

## 输入要求

### 必填输入
1. **PERILL诊断报告**：包含六维评分数据、问卷各题得分、健康评估、差距分析、盲点分析
2. **PERILL问卷评分数据**：原始评分表格，含团队现状评分和对一年后期待评分

### 可选输入
3. **团队背景信息**：团队成员介绍、组织背景、历史沿革、当前挑战等

## 语言风格要求

### 必须遵循
- **不出现具体评分数据**：不引用具体题号、不给出具体分数，用"评分相对偏低/较高""与其他维度相比""在六个维度中处于...水平"等相对性描述替代
- **不出现对具体团队成员的行为陈述和表现评价**：用角色化、功能化表述替代，如"团队领导者""资深成员""团队中的创新力量""具备调解凝聚作用的成员"
- **客观且积极**：用描述性和启发性语言揭示模式，每个维度的分析必须同时呈现积极面和改善空间
- **条理清晰自然**：以现象描述和模式分析为主，避免数据堆砌，让团队感到被理解而非被评判

### 语言风格指南

| 避免使用 | 推荐使用 |
|---------|---------|
| Q1评分4.5分 | 团队对共同目标的认知共识度相对较好 |
| Tom/Mary/Alice/Jack | 团队领导者/资深成员/调解凝聚型成员/创新力量 |
| 1.75分（全场最低） | 该维度在多个评估项中评分均相对偏低 |
| 差距1.97分 | 团队对该维度的改善有着较为强烈的内在诉求 |
| 失败/糟糕 | 改善空间/发展机会/可优化的方向 |
| 必须/应该 | 可以优先考虑/值得探索的方向 |
| 不信任 | 信任基础尚在建设中 |
| 冲突/矛盾 | 观点的多样性/不同视角的碰撞 |
| 领导者做错 | 领导风格与团队当前需求之间存在调适空间 |

**句式特点：**
- 多使用"值得关注的是...""这揭示了一个模式...""从整体来看..."
- 将改善空间表述为发展机会
- 用"同时""另一方面""在此基础上"等连接词平衡正反两面
- 每个维度分析以积极面开头，以发展契机结尾

## 诊断分析详细结构

### 第一步：读取输入材料
- 读取PERILL诊断报告和问卷评分数据
- 如有提供，读取团队背景信息
- 按需加载references目录下的参考文件

### 第二步：数据理解与分析（内部处理）
- 理解PERILL六维度评分数据（现状分、期望分、利益相关者分、差距值）
- 识别每个维度的相对优势和相对短板
- 识别团队自评与利益相关者评价的认知偏差
- 理解团队历史沿革和组织情境

### 第三步：生成六维度深度诊断分析

对每个维度（P, E, R, I, L, Ld），生成深度诊断分析，结构如下：

**【维度定位与概貌】（约80字）**
- 该维度在PERILL六维中的相对位置
- 用"处于较高/中等/较低水平""相对健康/存在改善空间/需要关注"等描述
- 一句话概括该维度的核心特征

**【积极面与优势基础】（约150字）**
- 该维度评分相对较高的方面，具体表现是什么
- 团队在哪些方面表现较好，这些优势的具体证据
- 这些优势对团队意味着什么，对团队未来发展有什么价值
- 这些积极因素如何支持团队在其他维度的发展

**【改善空间与深层议题】（约200字）**
- 该维度评分相对较低的方面，具体表现是什么
- 呈现为团队层面的模式和现象，而非个人问题
- 这些议题如何影响团队的日常运作
- 这些议题与其他维度的关联性
- 与团队历史、管理风格、组织环境等因素的关联

**【系统关联与影响路径】（约100字）**
- 该维度与其他维度的相互影响关系
- 该维度的状态如何影响团队整体系统
- 该维度的改善如何能带动其他维度的正向变化

**【发展契机与转化可能】（约100字）**
- 从现状与期望差距中看到的改进意愿
- 结合团队中的积极因素
- 这个维度改善后可能带来的连锁正向影响
- 改善的切入点和容易取得进展的方面

### 第四步：生成综合诊断分析（800-1000字）

从PERILL模型整体视角，生成系统性综合诊断分析：

**【整体图景与系统特征】（约150字）**
- 用概括性语言描述团队整体系统的特征
- 各维度的相对强弱分布形成怎样的整体图景
- 团队作为一个整体系统的核心特征
- 与团队背景和组织情境的关联

**【核心模式与负向循环】（约250字）**
- 识别团队当前陷入的系统级负向循环
- 用自然语言描述循环路径，说明"因为...所以..."的逻辑
- 各维度如何相互强化形成闭环
- 这个负向循环对团队整体效能的影响

**【关键杠杆与转化路径】（约250字）**
- 确定哪个维度作为干预杠杆最有效
- 理由：该维度评分相对较低、改善诉求最强、系统连接最广、可教练性最高
- 从负向循环转向正向循环的可能路径
- 改变这个维度能带动系统产生怎样的连锁反应

**【积极基础与前景展望】（约150字）**
- 团队已具备的积极基础和变革资源
- 团队成员展现出的发展意愿
- 对团队整体改善前景的积极展望
- 即使在最困难的维度也存在的成长潜力

### 第五步：生成团队发展建议（800-1000字）

基于诊断分析结论，生成连贯完整的发展建议：

**【整体方向与核心议题】（约100字）**
- 点明1-2个核心议题（对应诊断中的杠杆维度）
- 一句话概括整体发展目标和愿景
- 用过渡句引出下文："围绕这一核心，团队的发展行动可以沿着几个相互关联的方向有序推进"

**【方向一：关系与信任基础】（约200字）**
- 通常对应关系维度（R）或内部系统维度（I）
- 小标题引导："**在心理安全与信任建设方面**"
- 说明该方向解决什么问题、带来什么价值
- 结合诊断中的具体发现
- 2-3条具体可行动的建议

**【方向二：授权与协作机制】（约200字）**
- 通常对应内部授权（I/Ld）或外部协作（E）
- 小标题引导："**在内部授权与外部协作方面**"
- 说明该方向的杠杆效应
- 结合诊断中的具体发现
- 2-3条具体可行动的建议

**【方向三：愿景凝聚与学习成长】（约200字）**
- 通常对应愿景澄清（P）或学习机制（L）
- 小标题引导："**在愿景凝聚与持续学习方面**"
- 说明该方向如何固化成果
- 结合诊断中的具体发现
- 1-2条具体可行动的建议

**【实施节奏与阶段递进】（约150字）**
- 说明"先...再...然后..."的递进节奏
- 强调每一阶段为下一阶段创造条件
- 短期成效与长期目标的关系
- 指向团队自我教练的最终目标

## 质量检查清单

- [ ] 六维度诊断分析各500-600字，不出现具体评分和成员姓名评价
- [ ] 综合诊断分析800-1000字，用自然语言描述系统循环
- [ ] 团队发展建议800-1000字，包含核心议题、行动方向、具体建议、实施节奏
- [ ] 语言客观积极，描述现象和模式而非评判个人
- [ ] 诊断分析与团队背景中的积极因素相结合
- [ ] 发展建议与诊断分析结论一一对应
- [ ] 每个维度分析包含系统关联与影响路径部分
- [ ] 综合诊断明确指出杠杆维度和转化路径
