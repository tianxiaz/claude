# PERILL诊断分析框架

## 一、分析原则

### 1.1 客观性与积极性平衡
- 基于数据理解说话，但不输出具体评分
- 在揭示改善空间的同时，主动识别积极面和优势
- 避免评判性语言，使用描述性和启发性语言
- 让团队感到被理解而非被指责

### 1.2 系统视角
- 不孤立看待任何维度，关注维度间的相互影响
- 识别负向循环和正向循环，用自然语言描述而非符号
- 找到系统杠杆点——改变什么能带动最大系统效应

### 1.3 概括性表述（不出现具体评分和成员评价）
- 用"评分相对偏低/较高"替代具体分数
- 用"在六个维度中处于...水平"替代精确数值
- 用"团队成员对...的共识度较好"替代具体题号和评分
- 用"团队领导者""资深成员""新加入的成员""具备创新意识的成员"等角色化表述替代姓名
- 用"前任管理者遗留的管理风格"替代具体人名和历史描述
- 如需引用背景信息中的具体事实，用概括性语言转述

## 二、单维度分析结构（每维度500-600字）

### 结构模板

**【现状定位】**（1-2句，约50字）
- 该维度在PERILL六维中的相对位置
- 用"处于较高/中等/较低水平""相对健康/存在改善空间/需要关注"等描述

**【积极面与优势基础】**（2-3句，约150字）
- 该维度评分相对较高的方面
- 团队在哪些具体表现上值得肯定
- 这些优势对团队未来发展的积极意义

**【改善空间与核心议题】**（3-4句，约250字）
- 该维度评分相对较低的方面
- 呈现为团队层面的模式和现象
- 这些议题如何影响团队的日常运作和长期发展
- 与前序管理风格、组织环境等因素的关联

**【发展契机】**（2-3句，约150字）
- 从现状与期望差距中看到的改进意愿
- 结合团队中的积极因素
- 这个维度改善后可能带来的正向影响

## 三、综合诊断分析结构（600-800字）

### 3.1 系统性特征描述（约100字）
- 用概括性语言描述六维度的整体分布特征
- 例如："整体呈现出内部效能偏低而目标共识相对较好的不均衡格局"

### 3.2 负向循环识别（约250字）
- 识别当前团队陷入的系统级负向循环
- 用自然语言描述循环路径，不用"→"符号
- 说明各维度如何相互强化形成闭环

### 3.3 杠杆维度与转化路径（约250字）
- 确定哪个维度作为干预杠杆最有效
- 理由：相对最低评分、最大改善诉求、最强系统连接、最高可教练性
- 描绘从负向循环转向正向循环的可能路径

### 3.4 前景展望（约100字）
- 对团队整体改善前景的积极展望
- 强调团队已具备的积极基础和变革意愿

## 四、语言风格指南

### 4.1 使用词汇
| 避免使用 | 推荐使用 |
|---------|---------|
| Q1评分4.5分 | 团队对共同目标的认知共识度相对较好 |
| Tom/Mary/Alice/Jack | 团队领导者/资深成员/调解凝聚型成员/创新力量 |
| 1.75分（全场最低） | 该维度在多个评估项中评分均相对偏低 |
| 差距1.97分 | 团队对该维度的改善有着较为强烈的内在诉求 |
| 失败/糟糕 | 改善空间/发展机会/可优化的方向 |
| 必须/应该 | 可以优先考虑/值得探索的方向 |
| 不信任 | 信任基础尚在建设中 |
| 冲突/矛盾 | 观点的多样性/不同视角的碰撞 |
| 领导者做错 | 领导风格与团队当前需求之间存在调适空间 |

### 4.2 句式特点
- 多使用"值得关注的是...""这揭示了一个模式...""从整体来看..."
- 将改善空间表述为发展机会
- 用"同时""另一方面""在此基础上"等连接词平衡正反两面
- 每个维度分析以积极面开头，以发展契机结尾

## 五、字数控制要点

- 单维度分析控制在500-600字，约4-5个自然段
- 每段字数相对均衡，避免某段过长
- 综合诊断分析控制在600-800字，约4个自然段
- 用简洁自然的表达，避免冗长修饰
