# PERILL诊断分析框架

## 一、分析原则

### 1.1 客观性与积极性平衡
- 基于数据理解说话，但不输出具体评分
- 在揭示改善空间的同时，主动识别积极面和优势
- 避免评判性语言，使用描述性和启发性语言
- 让团队感到被理解而非被指责

### 1.2 系统视角
- 不孤立看待任何维度，关注维度间的相互影响
- 识别负向循环和正向循环，用自然语言描述而非符号
- 找到系统杠杆点——改变什么能带动最大系统效应
- 理解每个维度如何影响团队整体效能

### 1.3 概括性表述
- 用"评分相对偏低/较高"替代具体分数
- 用"在六个维度中处于...水平"替代精确数值
- 用"团队成员对...的共识度较好"替代具体题号和评分
- 用"团队领导者""资深成员""新加入的成员""具备创新意识的成员"等角色化表述替代姓名
- 用"前任管理者遗留的管理风格"替代具体人名和历史描述
- 如需引用背景信息中的具体事实，用概括性语言转述

## 二、单维度深度分析结构（每维度500-600字）

### 2.1 维度定位与概貌（约80字）

**目的**：建立对该维度整体情况的快速认知

**内容包括**：
- 该维度在PERILL六维中的相对位置
- 用"处于较高/中等/较低水平""相对健康/存在改善空间/需要关注"等描述
- 一句话概括该维度的核心特征

**示例**：
> "在六维评估中，'目的与动机'维度的现状评分处于中等偏上水平，表明团队对共同目标有基本的认知共识，但在深度和清晰度上仍有提升空间。"

### 2.2 积极面与优势基础（约150字）

**目的**：认可团队做得好的方面，建立信心

**内容包括**：
- 该维度评分相对较高的方面，具体表现是什么
- 团队在哪些方面表现较好，这些优势的具体证据
- 这些优势对团队意味着什么，对团队未来发展有什么价值
- 这些积极因素如何支持团队在其他维度的发展

**语言要求**：
- 描述具体现象而非泛泛表扬
- 说明优势与团队业务场景的关联
- 强调这些积极因素是可依赖的发展资源

**示例**：
> "值得肯定的是，团队成员对组织的长远使命有明确的认知，对共同目标的基本方向能够形成共识。在日常工作中，多数成员能够将个人目标与团队目标进行一定程度的协调。这种目标共识基础，为团队在未来进行更深入的愿景澄清提供了良好的起点。"

### 2.3 改善空间与深层议题（约200字）

**目的**：揭示需要关注的领域，但不造成指责感

**内容包括**：
- 该维度评分相对较低的方面，具体表现是什么
- 呈现为团队层面的模式和现象，而非个人问题
- 这些议题如何影响团队的日常运作
- 这些议题与其他维度的关联性
- 与团队历史、管理风格、组织环境等因素的关联

**语言要求**：
- 用"观察到一个模式..."而不是"问题在于..."
- 将个人表现上升为团队层面的模式
- 说明议题的深远影响，但不夸大其严重性
- 保持希望感，强调这是可改善的

**示例**：
> "在进一步深入观察中可以发现，团队在目标优先级的排序上存在一定的模糊性。当不同目标之间出现冲突时，团队缺乏明确的决策机制来协调优先级。这种状况导致了在资源分配和精力投入上的不确定性，有时会出现'眉毛胡子一把抓'的现象。进一步分析可以看到，这种模式与团队过往的决策习惯有关，也可能受到组织层面目标频繁调整的影响。"

### 2.4 系统关联与影响路径（约100字）

**目的**：展示系统思维，建立对维度间关系的理解

**内容包括**：
- 该维度与其他维度的相互影响关系
- 该维度的状态如何影响团队整体系统
- 该维度的改善如何能带动其他维度的正向变化

**语言要求**：
- 用"该维度与...维度形成相互影响的关系"而非箭头符号
- 描述影响路径的逻辑，而非简单罗列
- 强调联动效应的潜力

**示例**：
> "该维度与'关系'维度形成相互影响的关系。当团队对目标有更清晰的共识时，成员间的协作更加顺畅；而良好的人际关系又有助于在目标冲突时达成共识。这一正向循环的建立，将为团队整体效能的提升创造有利条件。"

### 2.5 发展契机与转化可能（约100字）

**目的**：为行动注入希望和方向

**内容包括**：
- 从现状与期望差距中看到的改进意愿
- 结合团队中的积极因素
- 这个维度改善后可能带来的连锁正向影响
- 改善的切入点和容易取得进展的方面

**语言要求**：
- 用"当...时，团队将能够..."的愿景式表述
- 具体说明改善后的积极后果
- 指出团队已有的可依赖资源

**示例**：
> "团队成员在访谈中表达了对目标更清晰化的渴望，这表明改进意愿已经在团队内部萌发。当团队建立起更系统的目标优先排序机制后，不仅能提升日常决策效率，还能增强成员对共同目标的责任感，从而为团队带来更充沛的集体能量。"

## 三、综合诊断分析结构（800-1000字）

### 3.1 整体图景与系统特征（约150字）

**目的**：建立对团队整体状况的认知框架

**内容包括**：
- 用概括性语言描述六维度的整体分布特征
- 团队作为一个整体系统的核心特征
- 与团队背景和组织情境的关联

**示例**：
> "从整体来看，该团队呈现出'内部效能有待提升，但目标方向相对清晰'的不均衡格局。六维度中，'目的与动机'和'学习'维度表现尚可，而'内部系统'和'关系'维度存在明显的改善需求。这种分布特征揭示了团队当前处于从'方向明确'向'执行有力'过渡的关键阶段。"

### 3.2 核心模式与负向循环（约250字）

**目的**：揭示团队陷入的问题循环，建立改变的紧迫感

**内容包括**：
- 识别团队当前陷入的系统级负向循环
- 用自然语言描述循环路径，说明"因为...所以..."的逻辑
- 各维度如何相互强化形成闭环
- 这个负向循环对团队整体效能的影响

**语言要求**：
- 用"当...时，往往会导致..."的句式
- 说明循环中每个环节如何引发下一个环节
- 强调这是可打破的系统模式，而非不可改变的个人特质

**示例**：
> "深入分析可以识别出一个持续强化的负向循环：由于内部决策流程不够清晰，团队成员在协作中经常需要等待明确指令；这种等待降低了协作效率，进而影响了成员间的信任；当信任受到影响时，成员在表达不同意见时变得更加谨慎；这种谨慎进一步削弱了决策质量，形成了'决策不清晰→等待指令→效率降低→信任受损→意见保守→决策质量下降'的闭环。"

### 3.3 关键杠杆与转化路径（约250字）

**目的**：指明干预的方向和路径

**内容包括**：
- 确定哪个维度作为干预杠杆最有效
- 理由：该维度评分相对较低、改善诉求最强、系统连接最广、可教练性最高
- 从负向循环转向正向循环的可能路径
- 改变这个维度能带动系统产生怎样的连锁反应

**语言要求**：
- 明确指出首选杠杆维度
- 解释选择该维度的理由
- 描述改变后的连锁效应

**示例**：
> "基于上述分析，'内部系统与流程'维度是干预的最佳杠杆点。选择该维度的理由是：它在当前评分中处于较低水平，成员对其改善有较强诉求，同时该维度与所有其他维度都有密切关联。当团队在内部流程和决策机制上取得实质改善后，将打破原有的负向循环，建立起'流程清晰→决策高效→信任增强→协作顺畅→流程进一步优化'的正向循环。"

### 3.4 积极基础与前景展望（约150字）

**目的**：注入信心，强调团队的变革潜力

**内容包括**：
- 团队已具备的积极基础和变革资源
- 团队成员展现出的发展意愿
- 对团队整体改善前景的积极展望
- 即使在最困难的维度也存在的成长潜力

**示例**：
> "值得欣慰的是，团队已经具备改变的重要基础：成员对共同目标有清晰认知，组织对团队发展有明确期望，团队内部存在强烈的进取意愿。这些积极因素，加上对问题模式的清醒认识，使我们有理由相信，通过系统性的教练干预，团队完全有能力走出当前的困境，迎来效能的显著提升。"

## 四、语言风格指南

### 4.1 词汇对照

| 避免使用 | 推荐使用 |
|---------|---------|
| Q1评分4.5分 | 团队对共同目标的认知共识度相对较好 |
| Tom/Mary/Alice/Jack | 团队领导者/资深成员/调解凝聚型成员/创新力量 |
| 1.75分（全场最低） | 该维度在多个评估项中评分均相对偏低 |
| 差距1.97分 | 团队对该维度的改善有着较为强烈的内在诉求 |
| 失败/糟糕 | 改善空间/发展机会/可优化的方向 |
| 必须/应该 | 可以优先考虑/值得探索的方向 |
| 不信任 | 信任基础尚在建设中 |
| 冲突/矛盾 | 观点的多样性/不同视角的碰撞 |
| 领导者做错 | 领导风格与团队当前需求之间存在调适空间 |

### 4.2 句式特点

- 多使用"值得关注的是...""这揭示了一个模式...""从整体来看..."
- 将改善空间表述为发展机会
- 用"同时""另一方面""在此基础上"等连接词平衡正反两面
- 每个维度分析以积极面开头，以发展契机结尾
- 用"当...时，团队将能够..."的愿景式句式

### 4.3 段落组织

- 每段聚焦一个核心观点
- 段落间用过渡句连接
- 自然段落长度约80-150字
- 避免过长的单一段落

## 五、字数控制要点

- 单维度分析控制在500-600字
- 包含5个明确的分析部分
- 每部分字数相对均衡
- 综合诊断分析控制在800-1000字
- 用简洁自然的表达，避免冗长修饰
