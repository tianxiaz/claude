# PERILL团队发展建议框架

## 一、发展建议原则

### 1.1 诊断驱动
- 发展建议必须基于诊断分析结论，从问题到建议有清晰转化逻辑
- 每个建议都对应诊断分析中识别的核心议题
- 避免与诊断无关的泛化建议
- 建议的具体方向应能从诊断中找到依据

### 1.2 聚焦与可操作
- 控制整体篇幅在800-1000字，要求精炼聚焦
- 核心议题1-2个，行动方向2-3个，具体建议3-5条
- 每条建议具体、可操作，与团队业务场景关联
- 避免面面俱到，优先解决杠杆维度问题

### 1.3 激励与引导
- 建议以激励团队成长与发展为目标
- 支持团队"借事修人"——在具体业务挑战中发展团队能力
- 为团队教练干预路径的有序推进提供支持
- 语气是支持和赋能，而非指令和要求

## 二、发展建议结构（800-1000字）

### 2.1 整体结构

发展建议输出为**一段连贯文字**，不分章节，融合核心议题、行动方向、具体建议和实施节奏。用加粗小标题引导不同方向，用自然段落和连接词组织内容。

### 2.2 段落组织

**开篇（约100字）**：
- 点明1-2个核心议题（对应诊断中的杠杆维度）
- 一句话概括整体发展目标
- 用"围绕这一核心，团队的发展行动可以沿着几个相互关联的方向有序推进"等过渡句引出下文

**方向一：基础层（约200字）**：
- 通常对应关系维度（R）或内部系统维度（I）
- 加粗小标题引导（如"**在心理安全与信任建设方面**"）
- 说明该方向解决什么问题、带来什么价值
- 结合诊断中的具体发现
- 2-3条具体可行动的建议

**方向二：核心层（约200字）**：
- 通常对应内部授权（I/Ld）或外部协作（E）
- 加粗小标题引导（如"**在内部授权与协作机制方面**"）
- 说明该方向的杠杆效应
- 结合诊断中的具体发现
- 2-3条具体可行动的建议

**方向三：升华层（约200字）**：
- 通常对应愿景澄清（P）或学习机制（L）
- 加粗小标题引导（如"**在愿景凝聚与持续学习方面**"）
- 说明该方向如何固化成果
- 结合诊断中的具体发现
- 1-2条具体可行动的建议

**结尾：实施节奏（约150字）**：
- 说明"先...再...然后..."的递进节奏
- 可用"四阶段"或"逐步推进"的表述
- 强调每一阶段为下一阶段创造条件
- 指向团队自我教练的最终目标

## 三、与诊断分析的衔接逻辑

### 3.1 从诊断到建议的转化路径

1. **诊断分析中的"杠杆维度"** → **发展建议中的"核心议题"**
   - 解释：干预杠杆维度通常是最有效的改变切入点

2. **诊断分析中的"负向循环"** → **发展建议中的"行动方向排序逻辑"**
   - 解释：打破负向循环需要从最关键的环节切入

3. **诊断分析中的"积极面与优势基础"** → **发展建议中的"积极基础"**
   - 解释：利用已有优势增强变革信心

4. **诊断分析中的"系统关联与影响路径"** → **发展建议中的"维度联动设计"**
   - 解释：一个维度的改善应能带动相关维度的正向变化

5. **诊断分析中的"发展契机"** → **发展建议中的"具体建议"**
   - 解释：改善的切入点也是发展的契机

### 3.2 建议的可行性检查

在提出建议时，应确认：
- 是否符合团队的实际资源约束
- 是否符合组织领导者的能力和风格
- 是否能在合理时间框架内见效
- 是否与组织整体战略方向一致
- 是否与团队成员的发展期望相符

## 四、具体建议撰写指南

### 4.1 建议的表述规范

- 不出现具体评分数据，用"该维度相对薄弱""改善诉求较强"等表述
- 不出现对具体成员的行为评价，用团队层面表述
- 不使用条目编号（"1.""2."），用自然段落组织
- 用加粗小标题区分方向（最多3-4个）
- 用积极正向的语言表述建议（"建立...""优化...""深化..."）
- 建议的语气是支持和赋能，而非指令和要求

### 4.2 好的建议特征

**具体性**：
- ❌ "加强沟通" → ✅ "每周五下午15:00-16:00举行15分钟的'工作同步会'，每位成员分享本周工作进展和下周计划"

**可操作性**：
- ❌ "提升团队凝聚力" → ✅ "每月组织一次团队建设活动，由轮值'活动官'负责策划和组织"

**关联性**：
- ❌ "加强培训" → ✅ "针对团队在跨部门协作中的薄弱环节，邀请协作部门同事进行1小时的业务介绍和经验分享"

**可验证性**：
- ❌ "增进理解" → ✅ "建立'伙伴观察制'，每周安排一位成员观察另一位成员的工作方式，并在周会上分享观察到的有效做法"

### 4.3 建议的语言模板

**引入建议时**：
- "值得考虑的做法是..."
- "一种有效的尝试是..."
- "可以考虑通过...来..."

**说明价值时**：
- "这将有助于..."
- "当...时，团队将能够..."
- "长远来看，这将为..."

**实施提醒时**：
- "需要注意的是..."
- "建议从...开始，逐步扩展到..."
- "在推进过程中，应关注..."

## 五、表述规范

### 5.1 词汇选择

| 避免使用 | 推荐使用 |
|---------|---------|
| 不能/禁止 | 建议考虑/值得探索 |
| 失败的问题 | 改善空间/发展机会 |
| 必须改变 | 值得优先调整 |
| 缺乏沟通 | 沟通方式有优化空间 |
| 信任缺失 | 信任基础尚在建设中 |
| 冲突未解决 | 不同观点的整合空间 |

### 5.2 句式特点

- 用"可以...""值得考虑..."等建议式句式
- 用"当...时，团队将..."的愿景式句式
- 用"这将有助于..."说明价值
- 用"需要注意..."进行适度提醒
- 避免命令式语气，多用邀请式语气

### 5.3 段落过渡

- 开篇到方向一："围绕这一核心，团队的发展行动可以沿着几个相互关联的方向有序推进。"
- 方向一到方向二："在奠定关系基础的同时，团队还需要在...方面建立更清晰的机制。"
- 方向二到方向三："当机制层面逐步完善后，团队可以进一步在...方面深化发展。"
- 到结尾："基于上述发展方向的逐步推进，团队可以按照以下节奏..."
