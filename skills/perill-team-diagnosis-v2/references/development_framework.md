# PERILL团队发展建议框架

## 一、发展建议原则

### 1.1 诊断驱动
- 发展建议必须基于诊断分析结论，从问题到建议有清晰转化逻辑
- 每个建议都对应诊断分析中识别的核心议题
- 避免与诊断无关的泛化建议

### 1.2 聚焦与可操作
- 控制整体篇幅在600-800字，要求精炼聚焦
- 核心议题1-2个，行动方向2-3个，具体建议3-5条
- 每条建议具体、可操作，与团队业务场景关联
- 避免面面俱到，优先解决杠杆维度问题

### 1.3 激励与引导
- 建议以激励团队成长与发展为目标
- 支持团队"借事修人"——在具体业务挑战中发展团队能力
- 为团队教练干预路径的有序推进提供支持

## 二、发展建议结构（一段式，600-800字）

### 2.1 整体结构

发展建议输出为**一段连贯文字**，不分章节，融合核心议题、行动方向、具体建议和实施节奏。用加粗小标题引导不同方向，用自然段落和连接词组织内容。

### 2.2 段落组织

**开篇（约80字）**：
- 点明1-2个核心议题（对应诊断中的杠杆维度）
- 一句话概括整体发展目标
- 用"围绕这一核心，团队的发展行动可以沿着几个相互关联的方向有序推进"等过渡句引出下文

**方向一：基础层（约180字）**：
- 通常对应关系维度（R）或内部系统维度（I）
- 加粗小标题引导（如"**在心理安全与信任建设方面**"）
- 说明该方向解决什么问题、带来什么价值
- 自然引出2-3条具体行动建议

**方向二：核心层（约180字）**：
- 通常对应内部授权（I/Ld）或外部协作（E）
- 加粗小标题引导
- 说明该方向的杠杆效应
- 自然引出2-3条具体行动建议

**方向三：升华层（约150字）**：
- 通常对应愿景澄清（P）或学习机制（L）
- 加粗小标题引导
- 说明该方向如何固化成果
- 自然引出1-2条具体行动建议

**结尾：实施节奏（约100字）**：
- 说明"先...再...然后..."的递进节奏
- 可用"四阶段"或"逐步推进"的表述
- 强调每一阶段为下一阶段创造条件
- 指向团队自我教练的最终目标

### 2.3 表述规范

- 不出现具体评分数据，用"该维度相对薄弱""改善诉求较强"等表述
- 不出现对具体成员的行为评价，用团队层面表述
- 不使用条目编号（"1.""2."），用自然段落组织
- 用加粗小标题区分方向（最多3-4个）
- 用积极正向的语言表述建议（"建立...""优化...""深化..."）
- 建议的语气是支持和赋能，而非指令和要求

## 三、与诊断分析的衔接逻辑

### 3.1 从诊断到建议的转化路径
1. 诊断分析中的"杠杆维度"→ 发展建议中的"核心议题"
2. 诊断分析中的"负向循环"→ 发展建议中的"行动方向排序逻辑"
3. 诊断分析中的"发展契机"→ 发展建议中的"积极基础"
4. 诊断分析中的"系统关联"→ 发展建议中的"维度联动设计"

### 3.2 建议的可行性检查
- 是否符合团队的实际资源约束
- 是否符合组织领导者的能力和风格
- 是否能在合理时间框架内见效
- 是否与组织整体战略方向一致

## 四、表述规范

- 不出现具体评分数据，用"该维度相对薄弱""改善诉求较强"等表述
- 不出现对具体成员的行为评价，用团队层面表述
- 用积极正向的语言表述建议（"建立...""优化...""深化..."）
- 建议的语气是支持和赋能，而非指令和要求
